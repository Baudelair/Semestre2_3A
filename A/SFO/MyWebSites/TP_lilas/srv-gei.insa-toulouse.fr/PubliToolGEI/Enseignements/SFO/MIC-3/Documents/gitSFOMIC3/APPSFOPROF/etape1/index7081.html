Comment marche le séquenceur d'un cœur?

Le cœur est composé d'un chemin des données (en bleu sur la figure 1.1) comportant les registres d'usage généraux (R0 à R15), l'ALU, le registre d'instruction (RI), le compteur ordinal (PC), etc. Ce chemin des données est commandé par l'unité de commande (UC en rose sur la figure) afin d'exécuter les instructions du programme exécutable binaire (byte code) situé en ROM périphérique (en vert sur la figure).

L'unité de commande (voir Polycopié chapitre 7), ou µ-séquenceur, est une machine à états finis synchrone :

  • un registre, le µPC, mémorise l'état de la machine sous forme d'une µ-adresse ;
  • une ROM, dite ROM de µ-code, est utilisée pour générer les µ-commandes correspondant à l'état du µ-séquenceur (signaux rouges)  ;
  • une logique combinatoire permet de déterminer le prochain état à partir des µ-commandes issues de la ROM µ-code (signaux rouges du haut), mais aussi en fonction d'entrées logiques issues du chemin des données (signaux bleus) et des périphériques (signaux verts).

Ainsi, sur chaque front montant de l'horloge, le séquenceur va changer d'état et envoyer les nouvelles µ-commandes (signaux rouges du bas) du chemin des données afin de faire progresser l'exécution d'une instruction d'assembleur : la ROM de µ-code contient donc un µ-programme correspondant à chaque instruction d'assembleur.

Chaque instruction machine (voir polycopié chapitre 5), contenue en général dans la ROM périphérique, est exécutée en 3 étapes :

  • fetch : qui consiste à aller chercher dans la mémoire l'instruction pointée par PC et la stocker dans le Registre d'Instruction (RI) ;
  • decode : reconnaître l'instruction et faire sauter la machine à état de l'UC vers le µ-programme d'exécution de l'instruction correspondante ;
  • execute : exécuter le µ-programme de l'instruction décodée puis retourner au fetch.

Fig 1.1 : Le cœur composé de l'unité de commande commandant le chemin des données agissant sur les périphériques. Les signaux bleus retournant du chemin des données vers l'UC permettent d'adapter la séquence de commandes à l'instruction à exécuter contenue dans le RI.
