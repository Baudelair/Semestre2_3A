Tutoriel

Le contenu de la mémoire peut être défini en utilisant le langage d'assemblage disponible avec le simulateur. Une ligne d'un programme écrit dans ce langage possède la syntaxe suivante (la syntaxe complète est indiquée dans la section intitulée BNF) :

ORG adresse  ; commentaires étiquette: EQU valeur  ; commentaires étiquette: WORD valeur  ; commentaires étiquette: COP paramètre1  ; commentaires étiquette: COP paramètre1, paramètre2 ; commentaires
ORG EQU et WORD sont des directives, COP est l'un des codes opérations présentés dans le sujet de l'étape 4.
  • ORG permet de spécifier l'adresse courante du prochain mot généré
  • EQU permet d'associer un nom (l'étiquette) à une valeur. Il n'y a pas de génération de mot mémoire.
  • WORD permet de définir un emplacement mémoire et sa valeur d'initialisation. L'étiquette permet d'associer un nom à l'adresse de cet emplacement mémoire.
  • COP permet de définir l'une des instructions de la machine (à une ou deux opérandes). L'étiquette optionnelle permet d'associer un nom à l'emplacement d'une instruction. Suivant le code opération, les paramètres utilisent l'un des modes d'adressage suivants:
    • registre : Ri avec i dans l'intervalle 0..7
    • immédiat8b : étiquette
    • immédiat16b : #valeur ou #étiquette
    • direct : [#valeur] ou [#étiquette]
    • indirect : [Ri] i dans l'intervalle 0..15
