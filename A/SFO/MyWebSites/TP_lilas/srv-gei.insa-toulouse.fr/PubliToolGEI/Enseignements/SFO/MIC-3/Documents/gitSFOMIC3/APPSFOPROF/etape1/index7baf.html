- 2.1 -

Maîtrise du µ-séquenceur

Récupérez les images mémoire etape1/RomMicrocode.mem, etape1/RomTranscode.mem et etape1/RomProgramme.ass en utilisant le commandes :

$ lilas /etape1/RomMicrocode.mem -extract $ lilas /etape1/RomTranscode.mem -extract $ lilas /etape1/RomProgramme.ass -extract

Lorsqu'on exécute dans un terminal la commande

$ lilas /arch/processeurComplet/ProcesseurComplet2015.lilas -awt=coeurAWT -script=initEtape1 -test
après avoir construit un modèle de la machine, le simulateur débute par l'exécution du script de nom initEtape1 défini dans le module arch.processeurComplet.ProcesseurComplet2015 .

Le module etape1/ProcesseurComplet2015_tst.lilas dispose également d'un script nommé exit. Ce script est utilisé automatiquement lorsqu'on quitte l'application. Il contient une commande permettant de sauvegarder automatiquement le contenu de la mémoire de microprogramme dans le fichier etape1/SaveRomMicrocode.mem si la mémoire a été modifié et non sauvegardée. Si un fichier de même nom existait déjà, il est conservé avec le nom etape1/etape1/SaveRomMicrocode.mem.save.aaaa-mm-jj-hh-mn-ss-msc (où aaaa-mm-jj-hh-mn-ss-msc correspond à la date de sauvegarde du fichier). Vous pouvez voir ces fichiers en utilisant la commande :

$ ls -a etape1

Votre premier travail consiste à µ-programmer le séquenceur de microprogramme (cf Polycopié Chapitre 7.3) pour obtenir la séquence de µ-instructions suivante :

Travail : Maîtriser le µ-séquenceur

Modifiez le µ-code pour obtenir la séquence ci-dessus, n'oubliez pas de sauver votre µ-code en cliquant le bouton save associé à la rom Microcode, pour ne pas perdre votre travail en cas de bug du simulateur (not zero-bug program).
