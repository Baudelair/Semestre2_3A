- 2.2 -

Spécifications du GPIO

Fig 2.1 : GPIO

Notre GPIO est une version 8 bits simplifiée du GPIO du STM32. Il est principalement composé de deux registres (à base de bascule FDCE) pouvant être chargés à partir d'une connexion 8 bits (Din) : CR (Control Register) et ODR (Output Data Register).

Travail : Maîtriser le GPIO

Pour ce travail, vous devez exécuter le module arch.gpio.GPIOTst en exécutant la commande

$ lilas /arch/gpio/GPIOTst.lilas -awt=awt
Vous obtiendrez alors la fenêtre suivante :

En utilisant ce programme test du GPIO, identifiez dans le schéma du circuit les registres CR et ODR.

Deux ports d'entrée/sorties sont connectés à des dispositifs : le port 2 à une Led, le port 4 à un switch. Quelles sont les valeurs que l'on doit mettre dans les registres CR et ODR pour :

  • lire la valeur du switch dans la sortie Dout
  • allumer la led (en rouge)
  • éteindre la led (en bleu)

Remarque: En écrivant la valeur 0x10 dans le registre CR, que constatez-vous quelle que soit la valeur positionnée par le switch ? Evidemment, une telle configuration devra être évitée dans la réalité, sous peine de détériorer de manière irréversible le connecteur associé dans le circuit FPGA de la carte Nexys3.
