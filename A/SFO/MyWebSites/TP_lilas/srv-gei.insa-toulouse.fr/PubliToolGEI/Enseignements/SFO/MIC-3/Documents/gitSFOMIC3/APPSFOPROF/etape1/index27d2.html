Maîtrise du simulateur

Pour ce projet, vous utiliserez un simulateur pour µ-programmer un cœur capable de s'initialiser et d'exécuter des programmes simples utilisant un jeu d'instructions machine.

Le lancement du simulateur s'effectue par l'exécution de la commande :

$ lilas /arch/processeurComplet/ProcesseurComplet2015.lilas -awt=coeurAWT -test -script=initSimuCtr
On obtient alors la fenêtre ci-dessous :
Le simulateur utilise une logique à 9 valeurs (celles définies par le standard IEEE 1164: ) :
  • U : Non initialisé
  • X : Inconnu
  • 0 : faux
  • 1 : vrai
  • Z : haute impédance
  • W : inconnu faible
  • L : faux faible
  • H : vrai faible
  • - : sans importance

Certains éléments graphiques permettent de modifier des valeurs. Lorsqu'il s'agit de valeur codée sur plusieurs bits (par exemple le contenu du registre microPC), la valeur apparait en caractères verts et soulignés. Il suffit de cliquer sur la valeur (elle apparait alors sur fond jaune), puis introduire une valeur et terminer par la touche return (le fond jaune disparait). Le format utilisé est le suivant :

  • 0x1234 pour une valeur en hexadécimal sur 16 bits
  • 0b001010 pour une valeur binaire sur 6 bits
  • 12:16 pour une valeur décimale (12) codée sur 16 bits
Lorsqu'il s'agit d'une valeur codée sur un seul bit, la valeur apparaît dans un cercle colorié suivant la valeur. Si la valeur est 0 ou 1, un simple cli permet d'inverser la valeur (de 0 on obtient 1, et de 1 on obtient 0). Si on veut obtenir une autre valeur, ou si la valeur n'est ni 0 ni 1 (par exemple U), on obtient un menu déroulant en cliquant sur le bouton tout en maintenant le bouton shift du clavier enfoncé. Il suffit de cliquer sur la valeur que l'on souhaite dans le menu pour la sélectionner.

L'interface graphique possède également des boutons particuliers (il y en a deux, mais vous pourrez en ajouter d'autres par la suite). Le bouton Reset permet d'exécuter un script d'initialisation (passage du signal reset successivement à 1 puis à 0). Le bouton Clk permet d'exécuter l'avancement du signal Clk sur une période (passge du signal d'horloge successivement à 1 puis à 0).

Travail : Que fait le microprogramme RomMicrocodeCtr.mem

Après avoir cliqué sur Reset, cliquez plusieurs fois sur le bouton Clock en examinant, à chaque microinstruction, les différentes commandes qui sont envoyées sur le chemin de donnée, et en regardant l'évolution des registres. Déterminez ce que fait la machine ainsi configurée.
